About Us

About IETE

The Institution of Electronics and Telecommunication Engineers (IETE) is India's leading recognized professional society devoted to the advancement of Science and Technology of Electronics, Telecommunication and IT. Government of India has recognized IETE as a Scientific and Industrial Research Organization (SIRO) and also notified as an educational Institution of national eminence. The objectives of IETE focus on advancing electro-technology. The IETE conducts and partners technical meetings, conferences, symposia, and exhibitions all over India, publishes technical journals and provides continuing education as well as career advancement opportunities to its members.

About AccessDenied

AccessDenied is a 36-hour hackathon organized by IETE-ISF for innovators across the country. It is an great opportunity to put your technical skills to use. Apply your textbook knowledge to real life ideas and stand a chance to win exclusive goodies, schwags, certificates and cash prizes upto Rs. 20,000. Mark your calendars for 23th and 24th March 2019.

Fortinet

FOR1 A website wants to track activity/sessions of its users and stores all that info in a Nosql database and there’re lakhs of such records. And there’s some other portion of user info that’s present in a different storage system (say an RDBMS database (that too single threaded and not a client server system) with much slower access rate in comparison to the one keeping the above dynamic data), and there are parts of the dashboard that fetch this info in conjunction with the dynamic data. Suppose that there’s a dashboard for administrators that shows a snapshot of the number of users, nature of activity (say watching certain shows available on the website, catching up some newsfeeds, etc)etc, whose monitoring is crucial per the business model. How would you ensure accesses to persistent data doesn’t become a bottleneck and slow up the activities on the dashboard. How would work you around the challenge posed by the database/storage system where certain ports of user info is kept and accessed as much as the other dynamic info present in the Nosql database

FOR2 Tracking a suspicious system activity.

a. We make a list few system resources, if user or any application tries to access or modify these resources, then alert the system admin via email.

b. Admin should get notification for each user login.

c. If too many failed log-in attempt from a particular client, inform the admin.

FOR3 String search problem: Design an algorithm to find a substring in a string in an efficient way, and compare it with Knuth Morris Pratt pattern matching algorithm. Develop the algorithm in C or C++ and test it with multiple samples of data

FOR4 Design an algorithm to enable an efficient file transfer to multiple clients, using only single file descriptor to access the file in the file system. Runtime memory usage should not cross file size.

FOR5 String search problem: Design an algorithm to find a substring in a string in an efficient way, and compare it with Knuth Morris Pratt pattern matching algorithm. Develop the algorithm in C or C++ and test it with multiple samples of data.

FOR6 Design and Develop an CPU and memory efficient stub for redis database, to achieve relational database functionality. Take sample table, of one lakh entries, with five fields and querable by any of the field values. The program should output the CPU and memory used used for each query. Efficiency of memory used should be output periodically, like memory fragmentation stats etc.

FOR7 Given a graph of access points(AP’s), nodes of graph representing AP’s and edges (undirected) between them representing ranges.

i.e. if two AP say AP1 and AP2 has edge between them meaning they are in range of each other.

We have to find minimum number of non-overlapping channels which we can assign to them.

Eg:

AP1--------------------------------------AP2-------------------AP3

Solution: Min 2 channels is required here.

Solution to above problem is a perfect example of graph colouring problem.

To solve it people have to find minimum number of colours which they can use to colour the graph so that no adjacent nodes have same colour.

Difficulty: Hard, this problem is NP-Hard, we have to go by heuristics to solve it.

DEF1-Design a surveillance system to support the activities for border security forces-Aim of any surveillance system is to provide observation into our Area of Responsibility and also to provide early warning. Elevated platform enhance the ranges as well as cover dead grounds, however the problem still persist.

    2. Expected Deliverables: To overcome shortcoming in present Battle Field Surveillance Radar. Extend surveillance ranges. Increase efficiency of existing systems.

  2. DEF2-Design a Prototype of Autonomous bot which can assist the troops during combat for equipment services- Sometimes soldiers step on mines and get serious injuries so the bot can detect the presence of land mines. During covert operations by commandos the bot can be used to view the inside activity of the conflict area without getting noticed by the enemy troops.

    3. Expected Deliverables: The solution should lay emphasis on how the bot can be used in the enhancement of defence technologies. It should guarantee the safety of the soldiers and provide assistance in battle and operations as well.

  3. DEF3-Identification of friend or foe (iff) system for armoured fighting vehicles IFF system is a tool of Combat Identification, “the process of attaining an accurate characterization of detected objects in the operational environment sufficient to support an engagement decision”. It also contributes to the overall tactical decision making.

    4. Expected Deliverables: To positively identify the opposing armoured vehicle prior to engagement. On battlefield, Armoured fighting vehicles would be required to engage enemy armoured fighting vehicles in time critical situations necessitating shortening of decision loop for engagement without positive identification of friend or foe which at times lead to fratricide by friendly fires.

Industrial automation
  1. IA1-Methods to deal with hazardous gases and possible leaks:Many industries deal with various hazardous gas. There is immense danger to all the workers if there happens to be a leakage. So it's very important to provide security to those employees working within these industries. If leakage of gas takes place then we need a system to provide alerts by turning ON an alarm which notifies the employers.

    Expected Deliverables: System to alert employees in case of a hazardous gas leak and identify the area of the leak.

  2. IA2- Atrocious lives of the traffic police: Nowadays the pollution in most of the metropolitan cities has hit unacceptable levels. Combining that with the increasing vehicular traffic, the life of traffic police has become a daylight atrocity with most being affected by the dust and pollution.

    Expected Deliverables: A suitable system to tackle the problems listed above as well as be economical.

  3. IA3-Damage Identification to belts or machinery: The constant use of heavy raw materials and equipment in industries causes the formation of cracks on industrial belts or even damage to the machinery.

    Expected Deliverables: A system to identify the points of cracks on the industrial belt. Further, even damage to machinery must be identified.

Health care

1.HC1-VR in Exposure Therapy and Treatment of anxiety disorders with VR.VR in Exposure Therapy and Treatment of Anxiety Disorders With VR as a deeply immersive and visual medium, it's not surprising to find its use in exposure therapy. Interactive VR platform in use at university centers, military bases, and veterans hospitals can offers patients virtual environments that mirror their traumatic experiences. A trained therapist controls the emotional stimuli and monitors their stress responses. Patient Recovery and Wellness using VR Many of these applications will benefit from the ongoing research in biofeedback monitoring. A highly innovative VR platform to map and respond to brain activity for applications in healthcare. VR is exploring ways to expand the effectiveness of using VR in therapy. A rehabilitation software motivates patients to perform specific movements, using games, movies, and 360 degrees interactive guides. What makes these applications incredibly valuable is the precise analytics they collect that can lead to further customization of patient care.

2.HC2-Diagnosis of Facial dysmorphic features using MLDiagnosis of facial dysmorphic features (ML) Facial recognition software is being combined with machine learning to help clinicians diagnose rare diseases (in this case, from facial dysmorphic features). Patient photos are analyzed using facial analysis and deep learning to detect phenotypes that correlate with rare genetic diseases.

3.HC3-Lack of access to quality clinical care close to the patient’s homes is the major cause for medical problems being detected late and having bad clinical outcomes. This is true for maternal care as well.In Primary Health Centres (PHCs) across India, clinical personnel are often ill equipped from a skill perspective to carry out timely interventions for expectant mothers. The doctors in a PHC follow an elementary checklist while treating pregnant women. Firstly, these checklists do not adhere to the standard recommendations of National Rural Health Mission(NRHM) or the World Health Organization(WHO) and hence medical conditions like risk of anaemia, Post-Partum Haemorrhage etc. are not detected early. Secondly, the doctors in PHCs only provide “minimal treatment” and refer most of the cases directly to the General Hospital. This causes even the most routine cases to be forwarded to the General/District hospitals instead of being handled in the PHC. Congestion in General/District hospitals due to this practice leads to their bandwidth not being available for critical cases. This project aims to provide a standardized care protocol that can self-enhance and learn contextually as doctors in PHCs use it.

Rural transformation
  1. RT1-Design a management system for effective distribution of water resources in rural areas based on the useOne of the major problems in rural areas is the effective distribution of water resources. While things such as agriculture need more water and regular lifestyle of people consume less water, distribution of water is not very proper in villages generally.

    Expected Deliverables: You are expected to come up with an effective management system which categorises and prioritises the requirements and allots the quantity of water effectively.

  2. RT2-Design a platform for controlling crime in the rural areas and effectively reporting the crime happening with peopleRural areas lack proper law and order. Victims find it difficult to report the instances happening with them and even after reporting such instances the proper action lacks. In some scenarios, nearby people can be of a great help.

    Expected Deliverables: You need to come up with an app which allows instant reporting and quick notifying the nearby people who can come to the scene and help. This should be simultaneously reported to the police station so that the police can reach the scene as soon as possible.

  3. RT3-Create a digital marketing network for sale of agricultural commodities for the benefit of farmer and end userFarmers these days face a lot of trouble in selling their stuff to the to the consumer. Farmers get exploited by the wholesalers in the market and do not get the deserved money as the intermediate sellers take a lot of the profit.

    Expected Deliverables: You need to design a platform where the farmers can go and sell their yield directly to the consumer who can receive the product at a much lower price as compared to market price as the intermediate commission is nullified.

  1. LT1-GPS Based Tracking Application with Logistics Management: To target this we develop new system which can provide real-time location of product and user (transport owner) can control sensor to maintain product freshness.Our system is GPS tracking system in which we track the truck which is supplied by supplier. Our system provides real-time asset tracking. In our system we have instant alerts on mobile SMS or Email to the truck driver to worn driver when there is some emergency or any defect in truck freezer. In this system we have required three hardware devices for tracking and measure the temperature of truck: -1. GPS module , 2. GSM module , 3. Temperature Sensor.

    Expected Deliverables:To develop an android app and website to Track And Maintain Freshness of Product.

Disaster
  1. DM1-Least affected path for evacuation:There is a lot of chaos during a disaster, people run here and there to find the best suitable place for their protection. The system should provide best path for them during such circumstances to reach a safe location.

    Expected Deliverables: Design a software/equipment to provide least affected and safe path to crowd for evacuation.

  2. DM2-Innovative devices for waste management after a natural disasterDuring Kerala floods after the flood water cleared off there were huge heaps of mixed wastes containing organic and inorganic matter which can be hazardous for the health and can even cause health problems for the flood survivors and can act as kickstart for various other diseases to emerge and spread. It even causes transportation problems due to blockage and makes it cumbersome for resources to be transferred from once point to other.

    Expected Deliverables:Design a model for removing the waste after a natural diaster

Society welfare
  1. SW1-Safety of school going kids Every now and then we hear about cases of young kids being kidnapped while commuting to and from school. This is a very serious problem that must be tackled. Since it is not suitable for young children to carry mobile phones, some other device or system must be developed for tracking and ensuring the safety of kids.

    Expected Deliverables:Design and develop an equipment/software for ensuring the safety of school kids travelling alone, with provision for SOS call.

  2. SW2-Application for hearing and speech impaired peopleThe life of a person who has both hearing and speech disability is very difficult. They cannot interact with everyone. Lip-reading is very difficult in come cases and sign language cannot be understood by most people. There should be a system that performs instantaneous text-to speech and speech-to-text conversion for making their lives easier.

    Expected Deliverables: Design and develop an equipment/software for making communication easier for hearing and speech impaired people.

  3. SW3-Pollution level analysis.Pollution is the new major environmental problem nowadays. With the increase in the number of respiratory diseases cases, people are becoming more conscious and aware about pollution as a major problem in today’s world. People generally avoid high pollution areas, or go there with proper precaution. However it would be much easier if there was a system that would determine the pollution level in their area (similar to live location based weather systems).

    Expected Deliverables: Design and develop an equipment/software for measuring the pollution level in the area near the user’s location.

FAQ
> Who is this hack for?
We encourage and welcome undergraduate students of all backgrounds to join us.
> What if I don’t know how to code?
Interest in learning and working with technology is much more important than your current experience level. No experience is needed — you will be able to work with various mentors, interact with companies, and learn alongside fellow participants.
> What if I don't have a team?
You can register individually and approach other teams if you would like to collaborate.
> Where is this?
The Hack will be at Vellore Institute of Technology, Vellore near Katpadi Railway station or a 2 and a hour hour drive from Chennai.
> What should I bring?
A valid student-id, a laptop and any hardware you'd require.
> How much does this cost?
Even though the price isn't set yet, we're trying to make it accessible to everyone.
